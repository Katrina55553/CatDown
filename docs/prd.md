# CatDown 产品需求文档（PRD）

## 1. Problem Statement

用户在 Windows 桌面上需要一个轻量、可高度自定义的“下班倒计时”小部件，能够：

- 直观显示距离下班/上班还剩多少时间；
- 自动处理跨夜、跨周末、节假日等复杂时间边界，避免显示负数或错误文案；
- 根据月薪实时估算“今天赚了多少钱”，增加趣味性与成就感；
- 支持字体、颜色、渐变、背景图片等视觉自定义，让桌面更个性。

目前市面上的倒计时工具要么不够美观，要么对工作日的处理过于简单（只按“当天下班时间”计算），无法满足真实办公场景。

## 2. Solution

**CatDown** 是一款 Windows 桌面小部件（Widget）。名称双关：**Countdown**（倒计时）+ **Off-duty**（下班）。

核心形态：程序常驻系统托盘，点击托盘图标后展开主窗口。主窗口内显示倒计时卡片与配置面板，并支持实时预览。

默认展示内容：

- 主倒计时（“下班还有 00:11:39” 或 “上班还有 2天 16小时”）
- 多个可开关的辅助信息卡片：发薪日、距离周五、下一个节日、今天收入
- 一只可替换/隐藏的猫咪主题插画

## 3. User Stories

1. 作为上班族，我希望点击托盘图标后看到距离下班的倒计时，以便掌握剩余工作时间。
2. 作为周五下午已下班的用户，我希望倒计时自动跳转到下周一上班时间，避免看到负数。
3. 作为调休/值班用户，我希望自定义哪些天是工作日，以便倒计时符合我的实际排班。
4. 作为弹性工作用户，我希望自定义上下班时间（如 9:30–18:30），而非被固定在 9:00–18:00。
5. 作为月薪制用户，我希望输入月薪后自动显示“今天赚了多少钱”，并按实际工作时长累计。
6. 作为发薪日固定的用户，我希望设置每月发薪日，并在发薪日落在周末/节假日时自动提前到最近工作日。
7. 作为视觉党，我希望字体支持渐变色，并能在独立弹窗中精细调整渐变。
8. 作为视觉党，我希望背景能使用本地图片，并支持裁剪，让小部件完美适配我的壁纸。
9. 作为谨慎的用户，我希望收入计算有明确提示说明公式，避免对数字产生误解。
10. 作为多设备用户，我希望配置能本地持久化，重启后保持设置。
11. 作为不想被打扰的用户，我希望组件可以隐藏猫咪插画或切换极简模式。
12. 作为中国大陆用户，我希望内置法定节假日，无需手动维护。

## 4. Implementation Decisions

### 4.1 桌面集成形态

- 程序以系统托盘应用方式运行，**点击托盘图标后展开主窗口**。
- 主窗口关闭后程序仍保留在托盘，右键托盘菜单可“显示主窗口”“开机自启”“退出”。
- 主窗口支持拖拽移动；展开后显示左侧实时预览区与右侧配置区。
- 支持开机自启开关；配置保存在本地 JSON 文件。

### 4.2 时间目标计算引擎

需要计算“下一个目标事件”及其文案：

| 当前状态 | 目标事件 | 显示文案 |
|---|---|---|
| 工作日，尚未上班 | 当天上班时间 | 距离上班还有 |
| 工作日，上班中 | 当天下班时间 | 下班还有 |
| 工作日，已下班 | 下一工作日上班时间 | 距离上班还有 |
| 非工作日 | 下一工作日上班时间 | 距离上班还有 |

关键规则：

- **工作日可配置**：默认周一到周五，用户可勾选任意组合。
- **工作时间可配置**：默认 09:00–18:00，精确到分钟。
- **下一工作日**：从当前时间向后查找，跳过非工作日；若用户未设置任何工作日，则给出配置提示，不显示倒计时。
- **跨夜/跨周末示例**：周五 17:00 仍按“下班还有 1小时”计算；周五 19:00 起目标变为下周一 09:00，显示“距离上班还有 2天 14小时”。
- **节假日**：内置中国大陆法定节假日数据，节假日视为非工作日，影响“下一工作日”与“发薪日提前”计算。用户可手动新增/删除自定义节假日。

### 4.3 收入计算引擎

公式：

```
todayIncome = monthlySalary / workDaysInMonth / workHoursPerDay × workedHoursToday
```

其中：

- `monthlySalary`：用户输入的月薪，非必填；未输入时不显示“今天收入”卡片。
- `workDaysInMonth`：按用户设置的工作日，统计**当前月份**的工作日天数（默认周一到周五，不含节假日）。
- `workHoursPerDay`：由配置的下班时间 − 上班时间自动得出。
- `workedHoursToday`：
  - 当天非工作日 → 0；
  - 工作日未上班 → 0；
  - 工作日上班中 → `min(now - startTime, workHoursPerDay)`；
  - 工作日已下班 → `workHoursPerDay`。

显示规则：

- 默认保留 **2 位小数**（如 ¥978.40）。
- 在设置中提供公式说明文案：“按当月工作日折算，未扣税/五险一金，仅供参考”。
- 当月薪未填写或工作时长为 0 时，卡片显示占位提示或自动隐藏。

### 4.4 发薪日逻辑

- 用户设置每月几号发薪（1–31 日）。
- 开启“遇节假日提前发”后：
  - 若发薪日落在非工作日（周末或节假日），向前寻找最近的工作日作为实际发薪日。
  - 若发薪日为 31 日而某月没有 31 日，则取该月最后一天。
- 倒计时显示“发薪日还有 X 天”。

### 4.5 配置面板交互

- **组件名称**：可自定义，默认“下班倒计时”。
- **工作日**：周一到周日点选标签，默认周一到周五。
- **工作时间**：两个时间选择器（时:分），默认 09:00 至 18:00。
- **字体颜色**：
  - 默认提供 8 个纯色圆点；
  - 最后一个圆点为“渐变”入口，点击后打开独立弹窗；
  - 弹窗内支持线性/径向渐变、角度、色标增删、实时预览。
- **背景**：
  - “颜色”模式：纯色或简单渐变；
  - “图片”模式：点击后打开本地文件选择（支持 JPG/PNG/WebP，建议限制 5MB）；
    - 选择后进入裁剪界面，支持固定比例（如 16:9、1:1）与自由裁剪；
    - 保存为本地文件路径或 Base64，视技术方案而定。
- **字体**：系统字体下拉框 + 自定义字体选择。
- **显示更多**：以开关标签形式呈现四项：发薪日、距离周五、下一个节日、今天收入。
- **发薪日设置**：每月几日 + “遇节假日提前发”开关。
- **月薪**：数字输入框，仅用于今天收入计算。
- **实时预览**：左侧实时渲染当前配置效果。

### 4.6 技术架构

- **桌面框架**：Electron。
- **前端框架**：推荐 Vue 3（Composition API），也可使用 React；本 PRD 不强制约束，由实现阶段决定。
- **状态管理**：Pinia（若用 Vue）或 Zustand（若用 React）。
- **日期处理**：使用成熟库（如 `date-fns` 或 `dayjs`），避免手写闰年/跨月逻辑。
- **持久化**：本地 JSON 文件，路径为用户数据目录（Electron `app.getPath('userData')`）。
- **节假日数据**：内置中国大陆法定节假日静态 JSON，支持热更新或用户手动编辑。
- **构建产物**：Windows 安装包（.exe）与便携版（.zip）。

## 5. Testing Decisions

- **时间引擎单元测试**：覆盖跨夜、跨周末、跨月、全周工作、无工作日配置、节假日等边界。
- **收入计算单元测试**：验证不同上下班时间、月薪、非工作日、已下班场景下的金额。
- **发薪日单元测试**：验证周末/节假日提前、31 日场景。
- **配置持久化测试**：验证读写本地 JSON、异常回退到默认配置。
- **UI 交互测试**：验证配置面板实时预览、渐变弹窗、图片裁剪流程。
- **本地化测试**：验证中文时间格式、货币符号、节假日名称显示。

## 6. Out of Scope

- 移动端适配。
- 多人协作/云端同步。
- 复杂考勤（午休、加班、迟到扣款）。
- 股票/基金等投资收益计算。
- 第三方日历深度集成（仅预留节假日接口）。
- macOS / Linux 版本。

## 7. Further Notes

- “今天收入”必须明确提示“未扣税、未扣五险一金、按工作日折算”，避免法律/财务纠纷。
- 背景图片裁剪建议在前端完成，减少后端依赖；最终保存方式取决于技术实现。
- 内置节假日数据需要每年更新，建议提供手动导入入口。
- 托盘图标应提供深色/浅色两套，适配不同任务栏主题。
- 若用户未配置任何工作日，主界面应友好提示“请先设置工作日”，而非显示错误。
